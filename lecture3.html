📂 File Path system

🔹 Step 1: The First Principle (The Fundamental Truth)

The most fundamental truth is this: A website is not one single file.
It's a collection of different files (HTML, CSS, JavaScript, images, videos, fonts) organized into folders.

🔹 Step 2: The Core Problem

If index.html needs logo.png, how does it know where to find it?
👉 A file path is the address of the file.

🟢 A) Relative File Paths (Most Important)

A relative file path gives directions starting from the current file’s location. You’ll use this 99% of the time.

my-website/
├── index.html
├── about.html
├── images/
│   ├── logo.png
│   └── hero.jpg
└── pages/
    ├── contact.html
    └── terms.html

✅ Scenarios

⭐ Scenario 1: SAME Folder
index.html → about.html

<a href="about.html">About Us</a>

✔ Just the filename.

⭐ Scenario 2: SUB-Folder (Going Down)
index.html → images/logo.png

<img src="images/logo.png" alt="Logo">

✔ Folder name + / + file.

⭐ Scenario 3: PARENT Folder (Going Up)
contact.html → images/logo.png

<img src="../images/logo.png" alt="Logo">

✔ ../ means go up one level, then down.

🟡 B) Absolute File Paths

Start with http:// or https://.
Use only for external files (Google, fonts, external images).

<a href="https://www.google.com">Search on Google</a>
<img src="https://upload.wikimedia.org/.../Html5_logo_and_wordmark.svg" alt="HTML5 Logo">

⚠️ Don’t use absolute paths from your computer (C:/...). They work only locally, not online.

🔒 Security Sandbox

Browsers keep your code in a "sandbox prison" so websites can’t read your private files.

📌 Summary / Rules of Thumb

  • 🟢 Own files → Relative paths
  • 🌐 External files → Absolute paths

💻 Windows vs macOS/Linux Absolute Paths

FeatureWindowsmacOS / Linux
Starting PointDrive Letter (C:\)Root /
Separator\ (Backslash)/ (Forward Slash)
ExampleC:\Users\John\Docs\file.txt/Users/john/Docs/file.txt

⚡ HTML Boilerplate Code

🔹 Step 1: The Principle

A browser needs setup info (doctype, language, encoding, viewport, title) before showing content.

🔹 Step 2: The Solution

We use a standard boilerplate — a universal starting template.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body>

</body>
</html>

✅ Why Use Boilerplate?

  • ✔ Avoids Quirks Mode
  • ✔ Prevents broken characters
  • ✔ Ensures mobile-friendly design
  • ✔ Helps SEO & Accessibility

🌐 Multipage Website

Complex sites → multiple pages (About, Services, Contact).
We need navigation + shared layout.

📦 The <div> (Generic Box)

Groups elements together. Example: profile card (image + name + bio).

🏷️ class & id (Labels)

  • id → unique (like Aadhaar Number)
  • class → reusable (like Job Title)

🖊️ <span> (Inline Highlighter)

Use for highlighting text inside a line → doesn’t break layout.

📑 Semantic Tags

  • 🔝 <header> → top section (logo, nav)
  • 📌 <main> → unique main content (only one per page)
  • 🔚 <footer> → bottom info (copyright, contact)
